# Repo Seed Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Populate engram wiki with AI/LLM concept, tool, and source notes seeded from Awesome-LLM and NicholasSpisak/second-brain.

**Architecture:** Fetch raw content from both repos via WebFetch, categorize each item as concept/tool/source, write notes using the engram templates, then update CLAUDE.md and index.md.

**Tech Stack:** WebFetch, markdown files, engram templates in `_templates/`

---

## Files Modified/Created

- Create: `wiki/concepts/*.md` — concept notes (techniques, papers, ideas)
- Create: `wiki/tools/*.md` — tool notes (frameworks, libraries, services)
- Create: `wiki/sources/*.md` — source notes (surveys, blog posts, articles)
- Create: `wiki/synthesis/*.md` — synthesis notes if second-brain has any worth keeping
- Modify: `CLAUDE.md` — add "Seeding from external repos" section
- Modify: `index.md` — append log entry

### Task 1: Fetch and parse Awesome-LLM

- [ ] **Step 1: Fetch the README**

  WebFetch: `https://raw.githubusercontent.com/Hannibal046/Awesome-LLM/main/README.md`

- [ ] **Step 2: Categorize entries**

  For each linked item in the README, assign a type:
  - Paper or technique (e.g., "Chain-of-Thought Prompting", "RLHF", "LoRA") → concept
  - Framework, library, or service (e.g., "LangChain", "vLLM", "llama.cpp") → tool
  - Survey, blog post, or article → source
  - Skip: items with no description, broken/missing links, non-AI-LLM content

- [ ] **Step 3: Commit fetch notes to memory**

  Keep a working list: `concepts[]`, `tools[]`, `sources[]` — each entry has a name, URL, and one-line description from the README.

### Task 2: Fetch NicholasSpisak/second-brain wiki files

- [ ] **Step 1: Fetch the repo file listing**

  WebFetch: `https://api.github.com/repos/NicholasSpisak/second-brain/git/trees/main?recursive=1`

  Look for files under `wiki/` with `.md` extension.

- [ ] **Step 2: Fetch each wiki file**

  For each `.md` file found under `wiki/concepts/`, `wiki/sources/`, `wiki/synthesis/`, fetch its raw content:
  `https://raw.githubusercontent.com/NicholasSpisak/second-brain/main/<path>`

- [ ] **Step 3: Filter to AI/LLM topics**

  Keep only files whose title or content is clearly AI/LLM related. Skip anything about productivity systems, personal development, unrelated tools, or domain-specific non-AI topics.

- [ ] **Step 4: Deduplicate against Awesome-LLM list**

  If a concept or tool from second-brain is already in the Awesome-LLM working list, merge: use whichever has the better description, keep all links. Don't create two notes for the same subject.
